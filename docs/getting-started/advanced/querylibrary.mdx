# Optional Command Parameter Library

This page is for players who want to get a little more precise with command usage. The bot supports several parameter styles across card, claim, and collection commands, and this reference focuses on the most common ones you are likely to run into.

:::note
Not every command uses every parameter. Many of these options show up most often in card-related flows such as /claim cards, /cards, /summon, /eval, and related collection commands.
:::

## Common parameter types

### card_query
Use this for card-based commands when you want to target a specific card or a group of cards.

Examples:
- `/cards card_query:twice`
- `/summon card_query:dark`

You can search by card name, collection, rarity, type, or other card traits. The bot will usually autocomplete the closest match, so you do not need to type the full name every time.

### Filtering a card group
When you want a subset of cards, you can add filters with `-` to include or `!` to exclude.

Examples:
- `-twice`
- `-enhypen`
- `-1`, `-2`, `!3`, `!4`
- `-promo`, `!lottery`, `-limited`, `!liveauction`
- `>levels`
- `-fav`, `>rating`

These filters are especially useful when you are trying to narrow a result list or search a very specific pool of cards.

### count
Use this when a command supports claiming or processing multiple cards at once.

Example:
- `count:5`

This is most relevant for claim-style commands such as /claim cards and /level claim.

### boost_id
If an event or boost is active, some claim commands can use a boost ID to increase the odds of getting a specific promo or event-related card.

Examples:
- `boost_id:christmas25`
- `boost_id:chuseok24`

### group_type
This is used when a claim command should only pull from a specific category of cards.

Examples:
- `group_type:girlsgroups`
- `group_type:boygroups`

Note that soloists are usually treated by gender rather than as their own separate group category.

### global
Use this when a command should look at the global card pool rather than only your personal inventory or the local guild context.

Examples:
- `global:true`
- `global:false`

## Quick reminder

If you are still getting comfortable with commands, the best approach is to start simple and let autocomplete help you. The advanced parameter syntax becomes much easier once you already know the basic command names and the kinds of cards you are trying to target.
